# 通用行为规则

## 先理解再动手
- 修改任何文件前，必须先阅读该文件的完整上下文
- 不确定接口签名、类型定义、函数参数时，先搜索源码确认，不要猜测
- 修改 Tauri command 时，同时检查前端 `invoke` 调用处和后端 `#[tauri::command]` 定义

## 最小化变更
- 只修改用户要求的内容，不要顺手重构、加注释、改格式
- 不要修改不相关的文件
- 不要添加用户没有要求的功能、依赖或错误处理
- 不要创建不必要的抽象、工具函数或辅助类

## 搜索优先
- 改代码前先搜索相关引用，确认影响范围
- 遇到不熟悉的模块，先用 Grep/Glob 了解结构，不要假设

## 输出纪律
- 所有对话、注释、commit message 使用中文
- 包管理只用 `bun`，禁止出现 npm/yarn/pnpm
- 版本号变更只能通过 `bun run vb`，禁止手动编辑
