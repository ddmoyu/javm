# JAVM v0.5.0

> `v0.4.2` → `v0.5.0`

## 新增

- 资源链接：记住上次选择的数据源，重启后保持；下载路径直接使用下载设置中的默认保存路径，无需每次手动选目录。
- 关于界面：新增「JAVM m3u8 Helper」浏览器脚本入口，方便从浏览器一键唤起 JAVM 下载。
- 设置：HTTP 失败回退 WebView 的开关移到通用设置区域，所有用户均可直接配置，不再需要开启开发者模式。

## 优化

- 启动体验：消除启动白屏与窗口位置跳动，修复显示卡顿（约 4 秒）问题，启动更顺滑。
- 媒体库：视频列表改用 shallowRef、搜索框加入防抖，上千条视频时滚动与输入明显更流畅。
- 下载管理：下载列表改为虚拟滚动，仅渲染可视行，任务上千时不再卡顿、内存占用大幅下降。
- 截取封面：截取的多张帧改为按视频时间顺序展示，第一帧固定在最前。
- 性能并发：截帧、预览图、封面图下载、目录扫描、刮削队列均改为有界并发，整体处理速度显著提升。
- 网络复用：刮削/翻译/图片下载的 HTTP 客户端改为进程级共享并复用连接池，减少重复 TLS 握手。
- 数据库：连接启用 WAL 与 busy_timeout，提升并发读写的健壮性，降低高并发下出错概率。
- 刮削稳定性：改进反爬检测（新增年龄验证门识别）与 WebView 回退策略，更新部分数据源的搜索地址。
- 依赖升级：升级 Rust 与前端依赖至最新版本，HTTP 统一走 wreq/BoringSSL 指纹方案。

## 修复

- 媒体库：修复下载到「目录管理」之外的视频也会出现在媒体库、且刷新无法消失的问题——现在媒体库仅展示目录管理内的视频。
- 下载管理：修复同一任务重复调度产生多进程/孤儿进程、开始/停止按钮忽略勾选、以及重启后未完成任务停滞不动的问题；同名任务自动加后缀避免下载到同一目录互相覆盖。
- 刮削：开始新搜索后，上一次搜索滞留的 WebView 弹窗现在会立即关闭，不再干等超时。
- 下载：修复下载失败时错误信息丢失的问题，失败原因可正常显示。
- 媒体探测：修复 ffmpeg 管道死锁导致部分多音轨/多字幕视频的时长、分辨率探测稳定失败的问题。
- 刮削写库：写库失败不再被误标为「已完成」，并改为单事务原子提交，避免出现数据不一致的中间态。
- 扫描：修复递归扫描跟随符号链接可能导致无限递归崩溃的问题。
- 设置：配置文件损坏时不再静默丢失全部配置，改为先备份为 `.json.bak` 并补齐字段级默认值，跨版本升级不再被重置。
- 解析器：修复多个数据源在处理外部 HTML 时可能因多字节字符切片导致的崩溃。
- 其他：AI 番号识别结果增加校验避免污染数据；深度链接添加任务时确保列表已加载并处理重复任务；下载文件名规避 Windows 保留设备名。

## 安全

- 新增 SSRF 防护：代理 HLS 请求仅允许 http/https 并拒绝本机/内网地址，防止探测内网服务。
- 下载任务入口清洗文件名，去除路径分隔符、`..` 与非法字符，防止路径穿越。

## 迁移说明

- 设置文件若损坏，升级后会自动备份为 `settings.json.bak` 再回退默认，不会再静默丢失全部配置。
- 无需用户手动操作的破坏性变更。
