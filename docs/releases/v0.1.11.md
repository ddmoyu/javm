# JAVM v0.1.11

## 概览

- 发布范围：`v0.1.10..HEAD`，并补充当前工作区中的 0.1.11 发布改动。
- 优化应用内更新流程，新增下载进度展示，并将“下载完成后立即安装”调整为“下载完成后由用户确认安装”。
- 修复更新说明来源，发布工作流会把正式发布日志写回 updater 的 `latest.json`，应用内更新弹窗可读取到更完整的版本说明。
- 重构 Tauri 端更新逻辑结构，将 updater 能力从入口文件中拆出，减少主入口膨胀，便于后续维护和继续扩展。

## 新增

- **更新下载进度显示**：应用内更新现在会展示下载百分比和已下载大小，用户可以更直观地判断更新包是否正在正常下载。
- **下载后确认安装**：更新包下载完成后，界面会进入“可安装”状态，由用户决定是否立即启动安装程序，避免下载一结束就自动退出应用。
- **更新状态细分**：新增下载中、待安装、启动安装器中的状态区分，设置页和更新弹窗会同步展示当前更新阶段。

## 优化

- **更新说明展示更完整**：Tauri 端补充 GitHub Release 正文兜底读取逻辑，GitHub Actions 发布流程也会把 `docs/releases/v0.1.11.md` 注入到 `latest.json`，减少应用内只看到空白更新说明的情况。
- **更新模块结构更清晰**：将 updater 相关逻辑拆分到独立模块，降低 `lib.rs` 复杂度，后续继续维护更新链路和发布逻辑时更容易定位问题。
- **发布工作流更闭环**：Release 创建完成后会回写 updater manifest，减少 GitHub Release 页面、应用内更新说明和实际发布内容不一致的问题。

## 修复

- 修复应用内更新按钮长时间处于 loading 时缺少进度反馈的问题，下载过程现在有明确状态显示。
- 修复更新包下载完成后直接启动安装、没有给用户确认机会的问题，避免误触后立刻退出应用。
- 修复 updater 指向错误仓库地址导致正式发布后更新源不一致的问题，统一为当前仓库的发布地址。
