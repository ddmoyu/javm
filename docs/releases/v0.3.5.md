# JAVM v0.3.5

> `v0.3.4` → `v0.3.5`

## 版本概览

- 媒体库与目录扫描链路继续优化，减少扫描过程中的数据库查询与缺失记录清理开销，大型库同步更顺畅。
- 媒体库页面新增手动刷新入口，扫描、导入或文件变更后可以更快重新拉取最新列表。
- 视频卡片与列表项的封面比例和高度计算进一步统一，浏览不同布局时显示更稳定。

## 主要更新

### 扫描与数据同步

- 扫描前会预加载已有视频的扫描信息，避免逐条回查数据库，提升目录扫描与结果汇总效率。
- 对已缺失的视频记录改为批量清理，减少无效条目残留在媒体库中的时间，也降低删除过程的额外开销。
- 目录页、媒体库页和资源刮削相关状态联动做了配套调整，扫描完成后的数据回显更连贯。

### 媒体库交互

- 媒体库页面增加刷新按钮，用户在扫描、导入或外部调整文件后，可以主动重新获取最新视频列表。

### 界面细节优化

- 调整了 VideoCard 和 VideoListItem 的封面样式，并优化 VirtualGrid 中的卡片高度计算，减少封面比例不一致时的视觉跳动。

## 使用说明

- 如果刚完成扫描、导入或手动整理了文件，媒体库内容未及时变化时，可以直接使用页面内的刷新按钮重新拉取列表。
- 重新扫描目录后，已经缺失或被移动的文件记录可能会被同步清理；如数量变化明显，建议核对一次目录实际内容。

## 风险与说明

- 本版本改动集中在扫描、缺失记录处理与媒体库刷新链路，建议重点验证大库扫描、文件移动/删除后的同步，以及刷新后的列表状态是否符合预期。
