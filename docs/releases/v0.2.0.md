# JAVM v0.2.0

## 概览

- 发布范围：`v0.1.14..HEAD`
- 新增代理配置模块，支持系统代理和自定义代理，所有 HTTP 请求统一走代理通道。
- 新增应用更新下载进度显示，更新过程可视化。
- 增强视频元数据管理，支持修改时间比较与封面版本控制。
- 视频文件自动归入独立同名目录，避免多视频共享资源目录。

## 新增

- **代理配置**：新增代理配置模块，支持从系统读取或用户自定义代理 URL；应用启动时初始化全局代理缓存，所有 HTTP 客户端（刮削、下载、更新检查等）统一应用代理设置；在设置页面保存代理配置后即时生效，无需重启应用。
- **更新进度显示**：应用更新下载和安装过程中显示实时进度条，更新按钮文本跟随状态变化；更新日志使用 marked + DOMPurify 安全渲染 Markdown 内容。
- **视频独立目录管理**：自动确保每个视频位于独立的同名目录中，避免多个视频共享同一资源目录导致的封面和预览图混乱；封面、预览图保存逻辑同步适配新路径。
- **视频库定时刷新**：添加视频库定时自动刷新功能，下载完成后自动触发视频库刷新，确保新内容及时可见。

## 优化

- **视频元数据增量更新**：数据库新增视频元数据修改时间字段，扫描时比较文件和 NFO 文件的修改时间，仅在变更时重新解析，减少不必要的全量扫描开销。
- **封面版本控制**：引入封面版本管理机制，封面文件变化时自动更新显示，解决缓存导致封面不刷新的问题。
- **正则表达式初始化优化**：将 `lazy_static` 替换为 `std::sync::OnceLock`，减少内存占用，提升下载进度解析和 ANSI 转义码处理的性能与可读性。

## 修复

- 无独立修复项，相关问题已在功能增强中一并解决。

## 兼容性与迁移

- 新增代理配置功能，默认为无代理，无需额外配置即可正常使用。
- 视频独立目录功能会在扫描时自动处理已有视频的路径规范化，无需手动迁移。
- 无特殊操作要求。
