# JAVM v0.3.6

> `v0.3.4` → `v0.3.6`

## 版本概览

- 媒体库与目录扫描链路持续优化，减少扫描过程中的数据库查询与缺失记录清理开销，大型库同步更顺畅。
- 媒体库新增手动刷新入口，视频卡片与列表项的封面比例和网格高度计算进一步统一，浏览体验更稳定。
- 补回批量截图封面任务创建入口，修复相关页面通过目录批量加任务时的前端调用缺失，也消除了该模块引起的构建阻塞。

## 主要更新

### 扫描与数据同步

- 扫描前会预加载已有视频的扫描信息，避免逐条回查数据库，提升目录扫描与结果汇总效率。
- 对已缺失的视频记录改为批量清理，减少无效条目在媒体库中的残留时间，也降低删除过程中的额外开销。
- 目录页、媒体库页和资源刮削相关状态联动做了配套调整，扫描完成后的数据回显更连贯。

### 媒体库与界面体验

- 媒体库页面增加刷新按钮，扫描、导入或外部调整文件后，可以主动重新获取最新视频列表。
- 调整了 VideoCard 和 VideoListItem 的封面样式，并优化 VirtualGrid 中的卡片高度计算，减少封面比例不一致时的视觉跳动。

### 截图封面与稳定性修复

- 批量截图封面页面恢复通过目录创建任务的前端调用，便于将无封面视频加入截图队列统一处理。
- 修复 resourceScrape store 中截图封面任务创建方法缺失的问题，避免相关功能调用失败，并消除构建阶段的 TypeScript 阻塞。

## 使用说明

- 如果刚完成扫描、导入或手动整理了文件，媒体库内容未及时变化时，可以直接使用页面内的刷新按钮重新拉取列表。
- 重新扫描目录后，已经缺失或被移动的文件记录可能会被同步清理；如数量变化明显，建议核对一次目录实际内容。
- 需要批量补截图封面时，可继续从目录入口将无封面视频加入截图任务列表，再统一执行处理。

## 风险与说明

- 本版发布日志按 `v0.3.4` 到当前发布前状态聚合整理，因此会同时覆盖 `v0.3.5` 已包含的功能更新与本次新增修复。
- 改动仍主要集中在扫描、媒体库刷新与截图封面链路，建议重点验证大库扫描、文件移动或删除后的同步，以及截图任务创建与执行流程是否符合预期。
