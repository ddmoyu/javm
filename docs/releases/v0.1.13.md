# JAVM v0.1.13

## 概览

- 发布范围：`v0.1.12..HEAD`
- 增强资源刮削流程，支持在遇到 Cloudflare 验证时自动弹出辅助 WebView，用户完成验证后继续执行刮削或视频链接查找。
- 优化刮削结果中的封面与预览图处理，补充远程预览图代理与 URL 规范化，减少搜索结果中图片已获取但界面不显示的情况。
- 改进视频文件扫描逻辑，增加对 `.ts` 文件的元数据探测，降低把 TypeScript 源文件误判为视频文件的概率。

## 新增

- **Cloudflare 验证辅助流程**：资源刮削和视频链接查找在遇到 Cloudflare 验证页时，会自动显示辅助 WebView，便于手动完成人机验证后继续流程。
- **远程预览图代理落地**：刮削结果新增远程预览图 URL 保留与代理写入能力，后续展示、保存和回看时可以更稳定地使用预览图资源。

## 优化

- **刮削图片显示稳定性**：统一规范化刮削结果中的图片 URL，并调整相关组件与状态流，减少封面、预览图已返回但界面展示不完整的问题。
- **Cloudflare 状态反馈更明确**：在刮削对话框、视频详情和视频链接查找界面增加验证状态提示，用户能更直观看到当前是否在等待手动验证。
- **视频扫描判断更准确**：将视频文件识别逻辑收敛到统一入口，并补充对 `.ts` 文件的探测逻辑，提升目录扫描结果的准确性。

## 修复

- 修复部分资源刮削结果中预览图资源已拿到但前端未正确显示的问题。
- 修复资源刮削或视频链接查找遇到 Cloudflare 验证时流程容易卡住、缺少明确引导的问题。
- 修复目录扫描时可能把非视频用途的 `.ts` 文件误识别为媒体文件的问题。

## 兼容性与迁移

- 无特殊操作要求。

## 已知问题

- 个别站点仍可能要求手动完成 Cloudflare 验证，且验证效果依赖当前网络环境与站点策略。
