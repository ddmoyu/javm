# JAVM v0.1.10-beta.4

## 概览

- 发布范围：`v0.1.10-beta.3..HEAD`
- 为旧版本本地数据库增加 schema 检测与重建流程，避免升级后沿用过期结构导致初始化失败或数据异常。
- 修复 GitHub Actions 发布时 updater 公钥覆盖不稳定的问题，降低测试版构建再次因 `failed to decode pubkey` 中断的风险。

## 新增

- **旧库版本识别**：应用启动时会检查本地 SQLite 的 `user_version`，识别出早期 schema 后自动删除旧库并重新初始化，减少跨版本升级时的兼容性问题。

## 优化

- **发布链路更稳健**：Tauri 打包阶段改为生成临时 release 配置文件后再执行构建，避免依赖环境变量覆盖默认 updater 占位配置，提升测试版发布成功率。
- **版本元数据同步**：桌面端 Rust 包版本与前端版本号同步到 `0.1.10-beta.4`，减少产物元信息不一致带来的排查成本。

## 修复

- 修复 Windows 发布工作流在某些情况下仍回退读取占位 updater 公钥，触发 `failed to decode pubkey` / `Invalid symbol 45` 并导致构建失败的问题。
- 修复旧版数据库升级到当前测试版时，仍保留低版本 schema 造成后续表结构与运行逻辑不匹配的问题。

## 兼容性与迁移

- 升级到此版本时，如检测到旧版本地数据库 schema，应用会自动删除旧数据库并重建。
- 如果旧数据库中仍有未迁移的数据，本次升级不会保留这些旧数据，需要在升级前自行备份。

## 已知问题

- 本版本仍依赖 GitHub Actions 中已配置的 updater 公钥、签名私钥与密码；如果这些仓库变量或 Secrets 缺失，发布构建仍会失败。
