# JAVM v0.1.10-beta.3

## 概览

- 发布范围：`v0.1.10-beta.2..HEAD`
- 修复 Windows 发布链路在生成 updater 产物时对 PEM 公钥处理不兼容的问题，避免因公钥格式带有头尾与换行而构建失败。
- 统一跨平台发布工作流文件命名，减少将其误认为仅服务 Windows 的维护成本。
- 仓库补充 MIT License，便于后续发布、分发与协作时明确许可信息。

## 新增

- **MIT License**：仓库新增 MIT License 文件，便于明确项目的开源授权范围与使用边界。

## 优化

- **Updater 公钥兼容性增强**：发布工作流现在会自动清洗 `TAURI_UPDATER_PUBKEY`，支持直接使用单行 base64 公钥，也支持从 PEM 公钥中剥离头尾与空白后再参与打包。
- **构建配置注入更稳健**：Tauri 打包阶段改为通过脚本生成 `TAURI_CONFIG` JSON，避免因手写字符串拼接导致公钥内容污染配置。
- **工作流命名更准确**：原本带有平台误导性的发布工作流文件已重命名为 `release.yml`，更符合其实际负责 Windows、macOS 与 Linux 的跨平台发布职责。

## 修复

- 修复 Windows 发布在生成 updater 相关产物时，因读取 PEM 形式公钥触发 `failed to decode pubkey` / `Invalid symbol 45` 并中断构建的问题。
- 修复同一套跨平台矩阵发布配置中的同类风险，避免后续 Linux 或 macOS 因直接消费原始公钥变量而出现相同失败模式。

## 兼容性与迁移

- 无特殊操作要求。
- 仓库变量 `TAURI_UPDATER_PUBKEY` 现在可使用单行 base64，或直接填写 PEM 公钥文本；工作流会在构建前自动规范化。

## 已知问题

- 正式发布仍依赖 GitHub Actions 中已配置的 updater 公钥、签名私钥与密码；若这些仓库变量或 Secrets 缺失，构建仍会失败。
