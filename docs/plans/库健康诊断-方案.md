# 功能方案:库健康诊断面板

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🟡 中(JvedioNext #3 标 🔴)
> 目标:聚合展示媒体库的元数据缺口与失败项(缺封面/NFO/演员头像、识别失败、刮削失败),并支持一键重试/补全。把散落的 `scan_status` 变成可操作的运维视图。

---

## 1. 现状

- 已有 `videos.scan_status`(0 待识别 / 1 已识别 / 2 已完成 / 3 识别失败 / 4 刮削失败)及各类 mtime 字段。
- 但**无聚合诊断视图**:用户无法一眼看到「多少缺封面、多少识别失败」,也没有批量重试入口。

## 2. 设计:诊断面板

聚合维度(各给计数 + 列表 + 一键操作):

| 诊断项 | 判定 | 一键操作 |
|---|---|---|
| 识别失败 | `scan_status=3` | 重新识别 / 手动指定番号(→ 非标准库方案) |
| 刮削失败 | `scan_status=4` | 重新刮削(可指定源) |
| 缺封面 | `poster`/`fanart` 均空 | 获取封面(→ 封面截图获取方案) |
| 缺 NFO | 无 .nfo / nfo_mtime 空 | 生成 NFO(→ NFO 方案) |
| 缺演员头像 | 关联演员 `avatar_path` 空 | 抓头像(→ 演员模块) |
| 缺截图 | thumbs 空 | 获取截图 |
| 元数据不完整 | 关键字段(标题/演员/日期)缺 | 重刮 / 字段融合补全 |

- 支持筛选 + 批量选中 + 批量执行(复用并发方案的任务进度)。
- 可选「健康分」:每个视频按缺口算完整度,库总览展示分布。

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `db/` | 诊断查询(按 scan_status + 字段空值聚合统计) |
| 后端 command | `get_library_health()`(各项计数)+ 分项列表查询 |
| 前端 | 诊断面板 view(卡片式各项计数 + 进入列表 + 批量操作) |
| 复用 | 现有重识别/重刮/封面/NFO/头像能力,诊断面板只是统一入口 + 批量编排 |

## 4. 分阶段

1. **统计查询 + 总览卡片**:各诊断项计数。
2. **分项列表 + 单项操作**:点进某项看具体视频,逐个修。
3. **批量操作 + 进度**:批量重试/补全。
4. **健康分 + 分布图**(可选)。

## 5. 风险

- 批量补全可能触发大量网络请求 → 复用反爬工具箱 + 并发限制。
- 判定规则需准确(避免误报「缺」)。
