# 功能方案:演员中心模块

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 目标:从零搭建独立「演员中心」——演员档案(头像+资料)+ 作品全集(本地有的 + 本地没有但能关联到的)+ 缺失作品一键刮削/下载。喜欢一个演员,即可快速获取其几乎全部信息与视频。

---

## 1. 现状

| 项 | 现状 |
|---|---|
| `actors` 表 | 已有 `id, name, avatar_path`;关联表 `video_actors(video_id, actor_id, priority)` |
| 资料字段 | ❌ 无身高/三围/生日/罩杯等 |
| 独立演员模块 | ❌ 无演员 command / store / view,前端仅在本地视频里按演员名筛选 |
| 源能力 | 各源能提取**演员名 + 演员页链接**(JavBus `.star-name a`、freejavbt `actor/{id}`),❌ 无「按演员列全部作品」 |

→ 头像字段已具备(缺填充);其余基本从零搭建。

## 2. 目标拆解

1. **独立演员模块**:演员列表页 + 演员详情页。
2. **演员档案**:头像 + 别名 + 身高/三围/罩杯/生日/出道日/国籍/简介。
3. **作品全集**:不止本地——抓取该演员的**全部作品列表**(分页爬完)。
4. **本地匹配**:作品全集 ∩ 本地库 → 标记「已有 / 缺失」。
5. **缺失获取**:缺失作品**仅补元数据**(封面/番号/标题)入库展示;**不自动下载**,下载由用户在影片详情页单独触发,演员页不批量下载。
6. **演员范围**:女优(有码)+ 无码演员 + 男优(全覆盖)。
7. **目标体验**:演员详情页 = 档案 + 全部作品(已有可播放、缺失有完整信息卡,可跳转影片页自行下载)。

## 3. 数据源能力(新增:按演员维度)

需要在刮削层新增「演员维度」三类能力(建议抽象 `ActorProvider`):
- `search_actor(name)` → 演员候选(各源演员 id + 名 + 头像)
- `get_actor_profile(actor_id)` → 资料(头像/身高/三围/生日…)
- `list_actor_works(actor_id, page)` → 作品列表(番号/标题/封面/日期),分页直到全集

实现来源:
| 用途 | 来源 | 说明 |
|---|---|---|
| **作品全集 + 头像 + 资料**(女优有码,主) | **JavBus `star/{code}` 页** | 一页含头像/资料/作品列表 + 分页(参考 hyperq/jav 的 `fetch_star_movies`、scraper.py 的 `avatar-box`);有码无码 URL 前缀区分(见 hyperq-jav.md) |
| **无码演员作品** | avsox / avmoo 等无码源 star 页 | 无码番号格式不同(见 pornboss.md 有码无码分轨) |
| **男优** | 男优源(参考 jav-moviemanager 的 AvdanyuScraper 思路) | 覆盖有限,作为补充维度 |
| 头像(兜底/补全) | **gfriends**(GitHub 社区头像库,FileTree.json 按名索引) | 见 jav-moviemanager.md |
| 资料(增强) | minnano-av(评分/身高/年龄/罩杯/三围) | 见 jav-moviemanager.md |
| 可选 | MetaTube `ActorProvider`(GFriends/XsList) | 见 metatube-sidecar 计划 |

**演员标识映射**:演员名 → 各源演员 id(JavBus star code 等)。多数已能从现有视频详情页的演员链接拿到(`star/{code}`、`actor/{id}`);拿不到则按名搜。存于 `actors.star_codes`(JSON)。注意重名/别名问题。

## 4. 数据模型设计

**`actors` 扩展字段**:`avatar_path`(已有)、`avatar_url`、`aliases`、`gender`(female/male)、`height`、`bust`、`waist`、`hip`、`cup`、`birthday`、`debut_date`、`nationality`、`blood_type`、`summary`、`work_count`、`profile_source`、`star_codes`(各源 id 映射 JSON)、`profile_updated_at`。

> `actor_works` 增加 `is_uncensored`(标记无码作品),区分有码/无码全集。

**新表 `actor_works`(演员作品全集)**:
```
actor_id     INTEGER  → actors(id)
code         TEXT     番号
title        TEXT
cover_url    TEXT
release_date TEXT
source       TEXT     来源
local_video_id TEXT   → videos(id),NULL=本地无
status       TEXT     local / missing / scraping / downloading / downloaded / failed
created_at / updated_at
UNIQUE(actor_id, code)
```

## 5. 核心流程

```
选/建演员
  │
  ├─① 解析 star_code(从已有视频演员链接 / 按名搜各源)
  ├─② 抓档案:JavBus star 页 → 头像+资料+作品全集(分页爬完)
  │        gfriends 补头像;minnano-av 补资料
  ├─③ 作品全集落 actor_works,与本地 videos 按番号匹配 → local / missing
  └─④ 缺失作品:**仅补元数据**(封面/番号/标题/日期)入库展示,**不自动下载**
         下载留给用户在影片详情页单独触发(复用 video_finder / 磁力)
         状态:missing(本地无文件,信息已补全)→ 用户在影片页下载完成后 → local
```

## 6. UI 设计

- **演员列表页**(新 view + 路由):头像网格,搜索 + 多维筛选(身高/罩杯/年龄/作品数/本地拥有数)。
- **演员详情页**:
  - 档案卡:大头像 + 资料(身高/三围/生日…)+ 作品总数/本地拥有数。
  - 作品网格:Tab「全部 / 本地有 / 缺失 / 有码 / 无码」;本地项可播放,缺失项显示封面/番号/标题,点击跳转影片详情页自行下载(演员页不批量下载)。
  - 批量操作:**补全所有缺失作品的元数据**(不含下载);进度展示(复用并发方案的任务进度)。

## 7. 改动 / 新增清单

| 层 | 改动 |
|---|---|
| `db/` | `actors` 扩展字段 + `actor_works` 表 + 迁移 |
| 新模块 `actor/` | commands(列表/详情/刷新档案/匹配/获取缺失)+ service(档案聚合 + 作品匹配 + 缺失编排) |
| `resource_scrape/` | 新增 `ActorProvider`:JavBus star 页(头像/资料/作品全集分页)、gfriends 头像、minnano-av 资料 |

| 前端 | `actor` store + 演员列表 view + 演员详情 view + 路由;复用并发任务进度 UI |
| `settings/` | 头像源/资料源开关、作品全集主源、缺失获取策略 |

## 8. 分阶段实施

1. **数据模型**:`actors` 扩展 + `actor_works` 表 + 迁移。
2. **演员档案抓取**:JavBus star 页头像+资料;gfriends 头像兜底。
3. **作品全集抓取 + 本地匹配**:分页爬全集,按番号标记 local/missing。
4. **演员列表 / 详情 UI**:档案 + 作品网格(已有/缺失)。
5. **缺失编排**:缺失 → 刮削 + 下载,状态机驱动(依赖快路径+WebView池、磁力方案)。
6. **批量 / 策略 / 设置**:获取策略、并发、筛选。

## 9. 风险与待确认

**风险**:
- **演员重名/别名**:同名不同人、艺名变更 → 需 star_code 为准 + 别名表。
- **作品全集准确性**:各源覆盖不同(有码/无码/素人);单主源会漏。
- **缺失全自动下载 = 海量**:一个演员可能数百部,带宽/存储/法律风险大 → 倾向手动或半自动确认,不默认全自动。
- 演员→star_code 映射失败的回退路径。
- 头像/资料版权(gfriends 等社区库),UI 呈现遵循项目「不显示真实站点名」约定。

**已决策**:
- ✅ 缺失获取:**仅补元数据,不自动下载**(下载由用户在影片详情页单独操作)。
- ✅ 演员范围:**女优(有码)+ 无码演员 + 男优**(需无码源 avsox/avmoo + 男优源;actors 加 `gender`,actor_works 加 `is_uncensored`)。

**仍待确认**:
- [ ] 作品全集主源:JavBus star(推荐),无码用 avsox/avmoo,是否再多源合并去重。
- [ ] 男优源具体选型(覆盖有限,可放到后期阶段)。
