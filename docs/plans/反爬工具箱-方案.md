# 功能方案:反爬工具箱

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🔴 高(javspider_stack 专文 + jabletv 印证)
> 目标:在现有 fetcher(wreq + TLS 指纹)之上,补齐代理池 / 镜像轮换 / 限速退避,提升所有爬取的稳定性与抗封禁能力,服务全部刮削与下载源。

---

## 1. 现状

- 已有:`fetcher`(wreq + TLS 指纹反爬)、`adaptive_concurrency.rs`(基于 CPU 负载的自适应并发,已领先)。
- 缺:智能代理池、多镜像域名轮换、请求限速、分级退避重试 —— 站点限频/封禁/改域名时缺乏韧性。

## 2. 工具箱组件(参考 javspider_stack `core/anti_block.py`)

| 组件 | 设计 |
|---|---|
| **UA 轮换** | UA 池,随机 / 轮询切换 |
| **成功率加权代理池** | 每个代理记成功/失败 → `success_rate`;按成功率**加权随机**选取;低于阈值(如 0.5)的代理临时剔除;失败自动降权 |
| **多镜像 URL 轮换 + 缓存** | 每个站点维护多个镜像域名(如 JableTV/fs1.app、JavBus 镜像);轮换使用,**缓存当前可用域名**到本地,失效自动切换 |
| **请求间隔限速** | 记 `last_request_time`,每次请求间随机延迟 `MIN~MAX`,不足则等待 → 礼貌爬取防封 |
| **分级退避重试** | 429 读 `Retry-After` 等待;其它错误 `5×attempt` / `10×attempt` 递增退避;最大重试次数可配 |

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `resource_scrape/fetcher.rs`(或新增 `anti_block` 模块)| 集成代理池 / UA 轮换 / 限速 / 退避;对外保持现有 fetch 接口 |
| 各源 `Source` | 声明镜像域名列表;URL 轮换器按站点取可用域名 |
| `settings/` | 代理列表、限速区间、重试次数、镜像开关 |
| 与并发协作 | 与 `adaptive_concurrency` 正交:并发控「同时几个」,限速控「单连接节奏」 |

## 4. 分阶段

1. **请求限速 + 分级退避**:最小改动、立即降低被限频概率。
2. **UA 轮换**:UA 池。
3. **多镜像 URL 轮换 + 缓存**:站点声明镜像,失效自动切换(对 JableTV/JavBus 价值大)。
4. **成功率加权代理池**:多代理时智能选优、自动避开失效代理。

## 5. 风险

- 代理质量参差 → 加权 + 阈值剔除可缓解;无代理时退化为直连。
- 限速降低吞吐 → 区间可配,按站点宽严不同。
- 镜像域名需维护(站点换域名时);缓存 + 探测自动适应。
