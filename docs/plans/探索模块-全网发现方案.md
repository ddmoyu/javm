# 功能方案:探索模块(全网发现)

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 定位:**独立于媒体库**的全网发现界面。媒体库管「本地有的」;探索管「全网的」——帮用户找喜欢的片子:搜女优 → 列出其全部作品 → 收藏 → 在线预览 → 好看才下载。

---

## 1. 架构分离:媒体库 vs 探索(关键)

| | 媒体库(已有) | 探索(本方案) |
|---|---|---|
| 数据 | **本地有的视频**(`videos` 表) | **全网数据**(爬取,不写入 `videos`) |
| 目的 | 管理/播放本地收藏 | 发现/筛选/收藏/预览,辅助决定下载 |
| 入口 | 主界面 | 独立「探索」入口 |
| 本地关系 | — | 探索结果里**本地已有的打标记**(可跳媒体库播放) |

→ **探索数据绝不污染媒体库**。`videos` 表仍只代表本地拥有。探索有自己的缓存与收藏表。

---

## 2. 核心流程

```
全网搜索(女优名 / 番号 / 片商 / 系列 / 分类)  或  从作品详情跳转某分面
   │
   ├─ 女优探索页:列出该女优【全部作品】(全网,分页拉全)
   │     每部标注:本地已有(可跳播放)/ 未拥有
   │
   ├─ 收藏:收藏女优 ★ / 收藏片子 ★(想看清单,本地可能没有)
   │
   ├─ 在线预览:播放该片在线流(m3u8),"好看才下载"
   │
   └─ 下载:对未拥有的片子发起下载(复用下载方案)→ 完成后进媒体库
```

## 3. 功能模块

### 3.1 全网搜索 / 分面跳转
- 搜索框:女优名 / 番号 / 关键词;结果分面切换(女优 / 片商 / 系列 / 分类)。
- 分面详情(复用「多维度浏览」框架的**探索型**模式):列出该分面的全网作品,分页。
- **女优探索页**:输入名字或从作品详情跳转 → 拉全该女优全部作品(全集型,分页拉全;复用演员模块的作品全集机制)。
- 交叉跳转:作品详情里的女优 / 片商 / 系列 / 分类都可点击进入对应探索页。

### 3.2 收藏系统(新)
- **收藏女优** ★:关注的女优,有新作可提示(呼应订阅)。
- **收藏片子 / 想看清单** ★:本地没有但想要的作品,作为后续下载的候选池。
- 收藏页:女优收藏 + 想看清单两个 Tab;想看清单项可批量发起下载。

### 3.3 在线预览(「好看才下载」)
- 预览该片的在线流(m3u8),前端 HLS 播放器(hls.js)或调用外部播放器流式播放。
- 片源复用在线视频获取(见 `视频获取-快路径与并发优化方案.md`:JableTV 明文 / MissAV surrit CDN 等)。
- 预览片段即可(不必整片),降低带宽;满意再下载。

### 3.4 下载入口
- 对未拥有作品发起下载:在线 m3u8 下载(N_m3u8DL-RE) / 磁力(见磁力方案)。
- 下载完成 → 文件入库 → 媒体库出现 → 探索里该片标记转「本地已有」。

## 4. 数据模型(探索独立存储,不动 videos)

- `favorite_actors`(收藏女优):actor 标识 + 名 + 头像 + 最近检查时间。
- `watchlist`(想看清单/收藏片子):番号 + 标题 + 封面 + 来源 + 加入时间 + 状态(wanted / downloading / owned)。
- `explore_cache`(探索结果缓存):分面类型 + 值 + 作品列表 JSON + 抓取时间(TTL,避免重复爬、加速翻页)。
- **本地匹配**:探索作品按番号 LEFT JOIN `videos` → 标「本地已有 + video_id」,只读,不写。

## 6. 风险与可行性

- **在线预览依赖在线站**(MissAV/Jable 等):有 CF/反爬/混淆,预览成功率与速度受其影响;失效需兜底提示。
- **全集准确性/覆盖**:女优全集来自源站(JavBus star 等),有码/无码/素人覆盖不同;探索缓存需 TTL 防过期。
- **下载/预览合规**:在线抓取与 P2P,维持「个人使用」定位;UI 遵循「不显示真实站点名」(以「数据源 N」呈现)。
- **数据量**:探索缓存可能膨胀,需 TTL + 上限清理。

## 7. 分阶段

1. **探索骨架**:独立「探索」入口 + 全网搜索(女优/番号)→ 结果列表(先不缓存)。
2. **女优探索页**:全部作品拉全 + 本地已有标记(复用演员模块全集机制)。
3. **收藏系统**:收藏女优 + 想看清单 + 收藏页。
4. **在线预览**:HLS 预览播放(复用快路径片源)。
5. **下载闭环**:想看清单/探索页发起下载 → 入库 → 标记转本地已有。
6. **分面扩展 + 缓存**:片商/系列/分类探索;explore_cache + TTL;女优新作提示。
