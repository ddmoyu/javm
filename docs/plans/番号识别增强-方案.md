# 功能方案:番号识别增强

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🟡 中(JvedioNext #1 标 🔴 + PornBoss / OpenAver / mediamatrix 多次涉及)
> 目标:扩充 `designation_recognizer.rs` 的识别覆盖面与鲁棒性,并保留版本/字幕/分片等语义标记。

---

## 1. 现状

`designation_recognizer.rs`:5 类正则(FC2-PPV / `ABC-123` / `T28-123` / `ABC123` 无连字符规范化 / `123456-789`)+ AI 兜底 + 校验(前缀 2-6 / 数字 3-8)。已较强,但有覆盖盲区与未保留的语义标记。

## 2. 增强项

### 2.1 覆盖面(新增格式)
- **素人**:`390JAC-132`、`300MAAN-783`(数字前缀 + 字母 + 数字)。
- **下划线无码**:`123456_999`、`HEYZO-1234`、`KIN8-1675`、`MYWIFE-1394`、`GETCHU-xxx`(与 `有码无码分轨-方案.md` 协同)。
- **后缀字母**:`SSIS-001A`(分片/版本后缀,参考 PornBoss `([a-z]{0,2})`)。
- **多分隔符**:连字符 / 下划线 / **空格**(`ABC 123`)。

### 2.2 语义标记(识别并保留,不当番号一部分)
| 标记 | 示例 | 含义 |
|---|---|---|
| 多碟 | `-cd1`/`-cd2`、`-A`/`-B` | 同片多文件 → 关联(配合作品-文件解耦) |
| 中文字幕 | `-C`、`-ch` | 字幕版 |
| 版本 | `UC`(无码破解)、`LEAK`(流出)、`4K` | 版本/规格 |
| VR | `_180_LR`、`_3dh`、`mkx200` | VR 文件名标记,保留不破坏番号 |

→ 番号归一(去标记得纯番号刮削),标记单独入库(展示/筛选/版本维度)。

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `utils/designation_recognizer.rs` | 新增正则集 + 标记提取(返回 番号 + 标记集) |
| `db/` | `videos` 加标记字段(字幕/版本/VR/分片),或复用 tags |
| 复用 | 现有 AI 兜底、校验、分辨率数字过滤 |

## 4. 分阶段

1. **覆盖面**:素人/下划线无码/后缀字母/空格分隔(扩正则 + 校验)。
2. **语义标记提取**:字幕/版本/VR/多碟,番号归一 + 标记入库。
3. **多碟关联**:`-cd1/-cd2` 归同一作品(配合作品-文件解耦,若做)。

## 5. 风险

- Rust regex 不支持 lookbehind → 用现有「校验 + 位置」机制替代边界否定环视(mediamatrix 用的 `(?<!...)`)。
- 纯数字无码易与分辨率/日期混淆 → 沿用并加强现有过滤(800/1080/720…)。
- 标记规则需保守,避免误删番号字符。
