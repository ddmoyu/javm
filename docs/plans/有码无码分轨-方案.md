# 功能方案:有码 / 无码 分轨

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🔴 高(3 次印证:PornBoss / hyperq-jav / NASSAV,OpenAver 旁证)
> 目标:从识别到刮削到展示,贯通区分有码 / 无码——无码番号格式不同、需专用源、URL 路径不同。当前各上层方案只用了「结果字段」,识别与来源层从未规划。

---

## 1. 现状

- 番号识别(`designation_recognizer.rs`)统一正则,不区分有码/无码。
- 各方案已用到「结果」:`actor_works.is_uncensored`、多维度 Tab「有码/无码」、别名方案 —— 但**识别层、无码专用源、URL 切换没有专门规划**(人人用、无人建)。

## 2. 设计(三层贯通)

### 2.1 识别层:双轨番号
- 有码:现有标准正则(`ABC-123` 等)。
- 无码:独立正则集 —— 纯数字 `010120-001` / `123456_999`、字母前缀无码(HEYZO/KIN8/MYWIFE/GETCHU)、FC2、素人。
- 输出 `is_uncensored` 标记(与 `番号识别增强-方案.md` 协同)。

### 2.2 来源层:无码专用源
- 无码作品走无码源:**avsox / avmoo** 等(参考 PornBoss `scanMissingUncensoredJavInfoWithAvsox`)。
- JavBus 有码/无码用 **URL `/uncensored` 前缀**切换(参考 hyperq-jav:列表 `/uncensored/...`、搜索 `type=1`、女优 `parent=uc/ce`)。
- 刮削编排:识别为无码 → 优先无码源;有码 → 常规源。

### 2.3 展示层(已规划,这里统一)
- `videos` / `actor_works` 的 `is_uncensored` 字段;多维度/演员/探索的「有码/无码」Tab 与筛选。
- **一键无码模式**:全局只用无码源(参考 OpenAver「一键切无碼模式」)。

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `utils/designation_recognizer.rs` | 增无码正则集 + 有码/无码判定 |
| `resource_scrape/sources/` | 接 avsox/avmoo 无码源;JavBus 源支持 `/uncensored` URL |
| `db/` | `videos.is_uncensored`(若无)+ 回填 |
| 编排 | 刮削队列按有码/无码选源 |
| 设置 | 无码模式开关、无码源启用 |

## 4. 分阶段

1. **识别双轨**:无码正则 + `is_uncensored` 标记(与番号识别增强一起做)。
2. **JavBus URL 前缀**:有码/无码列表/搜索切换。
3. **无码专用源**:接 avsox/avmoo。
4. **展示 + 一键无码模式**:Tab/筛选/全局开关。

## 5. 风险

- 无码番号格式杂(纯数字易和分辨率/日期混淆)→ 配合 `番号识别增强-方案.md` 的过滤与校验。
- 无码源覆盖与稳定性 → 配合反爬工具箱。
