# 功能方案:标准库 / 非标准库 双模式

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🟡 中(JvedioNext #2 标 🔴)
> 目标:番号识别失败的资源(合集、散装、无番号、特殊命名)不再被丢弃,进入「非标准库」单独管理,可手动归正。

---

## 1. 现状

- 识别流程番号驱动:`designation_recognizer` 识别失败(`scan_status=3`)即「识别失败」,无后续去处,资源等于被搁置/丢弃。
- 现实中大量资源无规范番号:多集合集、个人/素人散装、欧美/国产、改名文件。

## 2. 设计:两类库

| | 标准库 | 非标准库 |
|---|---|---|
| 进入条件 | 番号识别成功 | 识别失败 / 无番号 / 用户手动归类 |
| 元数据 | 正常刮削 | 不强制番号;可手动填标题/演员/标签,或手动指定番号后「转正」 |
| 浏览 | 正常维度浏览 | 独立列表,支持搜索/标签/手动整理 |

**关键:识别失败 → 自动进非标准库,不丢弃。** 用户可:
- 手动指定正确番号 → 重新识别刮削 → **转入标准库**。
- 不指定番号,仅手动打标题/演员/标签,留在非标准库管理。

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `db/` | `videos` 加 `library_type`(standard / non_standard);识别失败默认 non_standard |
| 后端 | 手动指定番号 command(→ 重识别/刮削/转正);非标准库手动编辑元数据 |
| 前端 | 非标准库视图(列表 + 手动整理 UI);从诊断面板「识别失败」可跳入 |
| 复用 | 现有识别/刮削/手动编辑;与诊断面板联动 |

## 4. 分阶段

1. **不丢弃**:识别失败标 `non_standard`,进独立列表(而非搁置)。
2. **手动指定番号 → 转正**:指定番号 → 重刮 → 转标准库。
3. **非标准库手动编辑**:手动填元数据/标签,独立浏览。

## 5. 风险

- 非标准库可能积累大量杂项 → 提供批量操作/隐藏。
- 「转正」需校验番号正确性。
