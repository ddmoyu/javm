# 功能方案:观看进度 / 继续观看

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🟢 低(JavHub #1、PornBoss 印证)
> 目标:记录每个视频的播放进度与历史,支持「继续观看」「最近播放」「已看」标记。

---

## 1. 现状

- 无任何播放进度/历史记录;播放后状态不留存。

## 2. 设计

### 2.1 数据
`videos` 增字段(或独立 `play_history` 表):
- `last_played_at`(最近播放时间)
- `position`(上次播放位置,秒)
- `duration`(已有)→ 算进度百分比
- `play_count`(播放次数)
- `finished`(是否看完,如 position/duration > 90%)

### 2.2 上报
- 播放器(内置/调用外部)在暂停/退出/定时上报当前位置 → 写 `position` + `last_played_at` + `play_count++`。
- 看完(接近结尾)标 `finished`。

### 2.3 展示
- **继续观看**:首页/媒体库置顶「有进度未看完」的视频,点击从 `position` 续播。
- **最近播放**:按 `last_played_at` 排序的快捷入口。
- **已看标记**:卡片角标;可按「已看/未看」筛选。
- 排序维度增加:播放次数、最近播放(呼应 PornBoss 的多维排序)。

## 3. 复用 / 改动

| 层 | 改动 |
|---|---|
| `db/` | `videos` 加播放字段 + 迁移 |
| 后端 command | `report_progress(video_id, position)`、`get_continue_watching()`、`get_recent_played()` |
| 前端 | 播放器上报;首页「继续观看/最近播放」区;卡片已看角标;筛选/排序 |

## 4. 分阶段

1. **进度记录 + 续播**:position 上报 + 从 position 续播。
2. **继续观看 / 最近播放 入口**。
3. **已看标记 + 播放次数排序/筛选**。

## 5. 风险

- 外部播放器(如 PotPlayer)进度回传困难 → 内置播放器易上报;外部播放器可退化为「记录已播放/次数」不记精确 position。
