# 功能方案:字段级跨源融合

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 优先级:🔴 高(3 次竞品印证:mediamatrix / PornBoss / OpenAver)
> 目标:多源刮削结果不再「整条择优只留一个源」,而是**按字段特长融合**,产出比任何单源都完整、准确的元数据。

---

## 1. 现状与问题

- 当前多源:并发抓各源 → 按 `detail_score` 排序 → **取最完整的单条**,其余源的字段**全部丢弃**。
- 问题:没有任何单源是全能的——社区源(JavBus/JavDB)有评分/分类,官方源(DMM)有权威标题/演员名,某些源封面更高清,中文源有中文简介。整条择优必然丢信息。
- MetaTube 同样是「整条择优」(见 `metatube-sidecar-集成计划.md` §4.3),不做字段融合 → 这是我们可做的差异化。

## 2. 设计:主源 + 按字段特长补全/覆盖

```
多源结果(已有 detail_score 排序)
  → 选主源(score 最高)作基底
  → 遍历其余源,按【字段优先级表】:
       · 主源缺失的字段 → 用其它源补
       · 关键字段 → 按 per-field 优先源覆盖
       · 数组字段(演员/标签/genre/thumbs)→ 并集去重
  → 融合成一条 ScrapeMetadata
```

### 字段优先级表(按字段信任不同源,可配置)
| 字段 | 优先源 | 理由 |
|---|---|---|
| 评分 / 分类 genres | 社区源(JavBus/JavDB) | 官方源无社区数据(参考 mediamatrix:rating/genres 恒留社区源) |
| 标题 / 演员名(官方) | 官方源(DMM)或日文源 | 权威、规范 |
| 中文标题 / 简介 | 中文源 | 本地化 |
| 封面 / 截图 | 高清源 / DMM 直拼 | 画质优先 |
| 系列 / 片商 / label | 任一非空,经规范化 | 配合 `跨语言别名与实体规范化-方案.md` |

## 3. 合并策略细节

- **标量字段**:优先源非空则用,否则按 score 降序取第一个非空。
- **数组字段**:并集 + 去重(演员/标签需先经别名规范化,避免「三上悠亜/三上悠亚」重复)。
- **冲突**:同字段多源不一致 → 按优先级表 + score 决定;记录来源便于回溯。
- **保留来源标注**:每条融合结果记录各字段来自哪个源(调试 / UI 可选展示「数据源 N」)。

## 4. 复用 / 改动

| 层 | 改动 |
|---|---|
| `resource_scrape/` | 在多源结果聚合处新增 `merge_sources(results)` → 融合;不改各源解析 |
| 配置 | per-field 优先源表(`settings`),给默认值 |
| 依赖 | 演员/片商名融合依赖 `跨语言别名与实体规范化-方案.md`(否则数组去重不准) |
| 复用 | 现有多源并发 + `detail_score` 排序(融合在其后) |

## 5. 分阶段

1. **基础融合**:主源 + 缺失字段补全(标量 + 数组并集),无 per-field 配置。
2. **字段优先级表**:评分/分类锁社区源、官方名锁官方源等,可配置。
3. **别名规范化接入**:数组去重前先规范化(依赖别名方案)。
4. **来源标注 + UI**:可选展示字段来源。

## 6. 风险

- 跨源演员/标签名不一致 → 必须配合别名规范化,否则并集出现重复。
- 字段冲突的优先级需合理默认,避免「劣源覆盖优源」。
- 融合增加少量计算(可忽略);保持与 `detail_score` 排序解耦。
