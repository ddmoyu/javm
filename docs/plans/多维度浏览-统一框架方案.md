# 功能方案:多维度(分面)浏览统一框架

> 状态:**规划中(未动代码)**
> 创建日期:2026-06-17
> 起因:需要片商(SSIS/IPX 等)模块、分类模块,与演员模块一致地「展示本地有的 + 本地没有的(带标记/过滤)」,并问能否三合一、还缺什么模块。

---

## 0. 核心洞察:演员/片商/系列/分类都是「分面」

演员、片商、系列、分类、导演、年份…本质是**同一种东西——分面维度**,都共享同一套交互:

```
分面值列表(所有演员 / 所有片商 / 所有分类)
   → 分面详情:该值的【作品全集】(本地有 + 本地没有,本地项有标记 / 可过滤只看本地)
      → 点作品看详情
         → 作品里的其它分面(演员 / 片商 / 系列 / 分类)可交叉跳转
```

所以不该各做一个孤立模块,而应做**一个统一的分面浏览框架**,演员/片商/分类只是不同分面。这直接回答了「三合一」。

> 术语澄清:**SSIS、IPX 是番号前缀 → 对应「系列/厂牌(series/label)」**,不是片商公司;**片商(maker)** 是 S1、IdeaPocket、SOD 等。本框架把两者都作为分面。

## 1. 现状

| 维度 | 存储现状 | 能否直接做模块 |
|---|---|---|

| 年份/月份 | 可从 `premiered` 派生 | ✅ 派生即可 |

---

## 2. 两种浏览模式(重要:不是所有分面都能「拉全集」)

**「本地没有的也显示」对不同分面可行性差异巨大**,数量级决定模式:

| 模式 | 适用分面 | 做法 |
|---|---|---|
| **全集型**(拉全 + 本地标记) | 演员、**系列/番号前缀(SSIS/IPX)**、小片商 | 一个系列/演员通常几十~几百部,可分页爬全集后本地匹配 |
| **探索型**(本地为主 + 按需翻页探索) | **大分类(genre)**、大片商(S1)、标签 | 一个分类可能上万部,**全集化不现实** → 本地全集 + 可选「探索全网」分页浏览(不强求拉全) |

→ 框架对每个分面声明其模式;UI 一致,但「缺失项」的来源策略不同(全集型主动拉全,探索型按需翻页)。

---

## 3. 统一界面设计(回答「三合一」)

**能,而且应该。** 一个「发现 / 浏览」界面:

- **顶部分面切换**:演员 | 片商 | 系列 | 分类 | 导演 | 年份(可扩展)。
- **分面值列表**(通用组件):网格/列表,搜索 + 排序(作品数 / 本地拥有数 / 名称);演员/片商带头像/Logo。
- **分面详情**(通用组件):头部信息卡(演员=档案,片商/系列=名称+作品数)+ **作品网格**。
- **作品网格**(通用组件,全模块复用):
  - Tab/过滤:全部 / **仅本地有** / 缺失 /(有码 / 无码)。
  - 本地项有「已有」标记可播放;缺失项显示封面/番号/标题。
  - **全局过滤开关**:「只显示本地拥有」(你要的过滤选项,所有分面通用)。
- **交叉跳转**:作品详情里的演员/片商/系列/分类都是链接,点击进入对应分面详情。

→ 三个模块复用同一套「分面值列表 + 分面详情 + 作品网格 + 本地过滤」组件,只是数据源(分面类型)不同。

---

## 4. 数据模型

沿用现有 actors/tags/genres 的「独立表 + 关联表」模式,补齐缺失维度:

- 新增独立表 + 关联:`studios`(制作商/片商)、`series`(系列/厂牌,**可由番号前缀自动建立**)、`directors`;各配 `video_studios` / `video_series` / `video_directors`。
- **系列/番号前缀**:从 `local_id`(如 `SSIS-001`)提取字母前缀 `SSIS` → `series`。扫描/刮削时自动归类。
- 年份/月份:从 `premiered` 派生,无需建表。
- `actor_works`(演员模块已规划)可推广为通用 `facet_works`(分面→作品全集 + 本地匹配 + 缺失标记),供全集型分面共用。

> 备选:通用 `facets(type, value)` + `video_facets` 单表方案,加维度不改 schema;但与现有 actors/tags/genres 风格不一致。建议主维度沿用独立表,保持一致。

---

## 5. 还缺什么模块(你问的)

**分面类(纳入本框架)**:
- ✅ 演员(已规划)、片商、系列(SSIS/IPX)、分类/标签
- 🆕 **导演**、**发行年份/月份**(时间线浏览)

**非分面类(独立模块,建议补)**:
- **想看 / 收藏清单(Watchlist)**:跨演员/片商收藏「本地没有、想要的作品」,配合下载。
- **排行 / 榜单**:评分榜、今日/本周热门(探索型,来自源站榜单页,呼应 missav 的 dmXXX 分类)。
- **最近添加 / 最近播放 / 随机推荐**:本地库的快捷入口(呼应 PornBoss/JavHub 的继续观看)。
- **诊断/缺口面板**(见 jvedionext 分析):缺封面/NFO/演员头像的聚合视图。

## 7. 分阶段

1. **维度独立化**:studio/series/director 建表 + 番号前缀→series 归类 + 关联回填(从存量 videos)。
2. **通用分面组件**:分面值列表 + 详情 + 作品网格 + 本地过滤(先用本地数据跑通)。
3. **分类/片商/系列 本地模块**:基于已入库数据(探索型先只做本地)。
4. **全集化**:系列/片商接「作品全集抓取 + 本地标记」(复用演员模块机制)。
5. **统一「发现」界面**:分面切换整合三合一;交叉跳转。
6. **补充模块**:想看清单 / 榜单 / 最近 / 诊断面板。
