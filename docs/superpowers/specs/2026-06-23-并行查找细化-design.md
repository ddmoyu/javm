# 并行视频源查找 — 细化 设计文档

日期：2026-06-23
状态：已确认，直接实现（用户授权跳过单独 plan/SDD 编排）

承接 `2026-06-23-并行视频源查找-design.md`。三项细化：

## 1. UI 改为「平铺正片链接列表」，不列下载源

- 调度器（`useParallelFinder.ts`）**不动**：仍后台 3 路并行、每源出正片即停接力。
- `VideoLinkFinder.vue` 从 `scheduler.sources` 派生 `foundLinks`：取 `status==='found' && realLink` 的源，映射为 `{ ...realLink, siteId }`，**按 url 去重**。源一出正片就追加进列表。
- 模板：删除"每源一行（状态徽章）"那块。改为：
  - 顶栏：番号输入 + 查找/停止 + **一行轻量进度**（转圈 + `已找到 N 个链接`；**不列源、不显示每源成功/失败**）。
  - 主体：平铺 `foundLinks`，每条 = 资源名（脱敏 `资源 N`/开发者模式真名）+ 时长/分辨率徽章 + 链接 + 预览/下载。
  - 底部：批量下载（所有正片）+ 保存路径。
  - 空/搜索态：`running` 且无链接时显示"正在并行查找…"。
- 保留：下载、查重、预览、复制、保存路径解析、`autoSearch` 暴露。

## 2. 单源超时 120s → 60s

- 设置默认 `linkFinderSourceTimeoutSecs`：后端 `default_link_finder_source_timeout()` 120→60；前端 `defaultSettings.scrape.linkFinderSourceTimeoutSecs` 120→60。clamp 下限保持 30、上限 600 不变。

## 3. 每源「失败检测规则」：CSS 选择器 + 可选文本包含

立即判失败、不等 60s 超时。命中即 emit `page-state {site,'not-found'}`。

- **源定义结构化**（`video_finder.rs`）：`DEFAULT_DOWNLOAD_SITES` 从 3 元组 `(id,name,url)` 改为结构体常量：
  ```rust
  pub struct DownloadSiteDef {
      pub id: &'static str,
      pub name: &'static str,
      pub url_template: &'static str,
      pub fail_selector: &'static str, // 空串=无规则
      pub fail_text: &'static str,     // 空串=只判选择器命中；非空=元素 textContent 还需含该串
  }
  ```
  `build_site_url` 与 `rs_get_video_sites` 随之改（按 `.id`/`.url_template`/`.name` 取值，逻辑不变）。新增 `site_fail_rule(site_id) -> (&str, &str)` 取该源规则（找不到返回空）。
- **注入**：`INTERCEPT_JS` 顶部加 `var __FAIL_SEL__ = '__VIDEO_FINDER_FAIL_SEL__'; var __FAIL_TEXT__ = '__VIDEO_FINDER_FAIL_TEXT__';`；`open_video_finder_webview` 注入前对脚本做 `.replace("__VIDEO_FINDER_FAIL_SEL__", sel).replace("__VIDEO_FINDER_FAIL_TEXT__", text)`（与现有 `__VIDEO_FINDER_SITE__` 同处、同样两处注入点都替换）。占位符注意转义：选择器/文本里一般无单引号，若有需转义（实现时用 JSON 字符串拼接更稳，见下）。
  - 更稳的注入：用 `serde_json` 把 sel/text 序列化成 JS 字符串字面量再替换，避免选择器含引号破坏脚本。
- **检测**：在 `INTERCEPT_JS` 的 `reportNotFound()` 里，通用 `NOT_FOUND_RE`（title/h1）**之外**追加：
  ```js
  if (__FAIL_SEL__) {
      var el = document.querySelector(__FAIL_SEL__);
      if (el && (!__FAIL_TEXT__ || (el.textContent || '').indexOf(__FAIL_TEXT__) !== -1)) {
          /* emit not-found（同现有逻辑） */
      }
  }
  ```
  随周期扫描（首次 1s、之后每 3s）执行，命中即上报。通用正则保留作没配规则的源的兜底。
- **规则是数据，增量填**：初始大多 `fail_selector: ""`（仅走通用兜底）；按各站 404 页逐个补。先填能确定的（无把握的留空，不臆造）。

## 边界 / 测试

- 去重：`foundLinks` 按 url 去重（不同源同一 m3u8 罕见，去重更干净）。
- 规则为空 → 跳过该检查，仅通用正则兜底；不影响现有行为。
- 注入字符串安全：用 JSON 序列化 sel/text 成 JS 字面量。
- 测试：`foundLinks` 派生逻辑可加一个轻量纯函数单测（输入 sources → 输出去重正片列表）；注入 JS 与 UI 主要靠真机手验。`pickRealLink`/调度器不变。

## 不做（YAGNI）

- XPath（CSS 选择器 + 文本包含已够）。
- 列全部候选链接（只列正片）。
- 失败规则的可视化配置 UI（先硬编码在源定义里）。
